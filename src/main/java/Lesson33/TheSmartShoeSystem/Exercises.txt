Exercise 5 – Reflection (thinking task)
    Question: Why is it useful that we can switch validators without changing Shoe?
    Answer: Because the validate()-method can be polymorphed depending on the validator and which exception it throws,
    but also honors the Open/Closed principle of SOLID (add functionality without having to modify the existing code).

    Question: What would break if validation was hardcoded inside Shoe?
    Answer: The Single Responsibility and Open/Closed principles of SOLID. Furthermore, it would not be able to reuse
    the validation logic anywhere else. You would have to change directly in Shoe.java everytime you'd want to add
    another type of shoe (using e.g. if/else trees).
